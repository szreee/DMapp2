ここから先は、**LLMで22,000枚超のカードを取り込むための本丸**です。

実際には、

```text
KAFS
=
キーワード能力辞書

ATC
=
自由記述能力辞書
```

です。

デュエマのカードテキストを大量に分析すると、

```text
完全に固有能力
```

は意外と少なく、

多くは

```text
カードを1枚引く

相手のクリーチャーを1体破壊する

山札の上から○枚見る

墓地から回収する

パワーを+○○する

条件を満たしたら召喚する
```

などのテンプレートの組み合わせです。

そのためATCは

```text
自然言語
↓
能力テンプレート
↓
DSL
↓
Bytecode
```

へ変換するための辞書になります。

---

# Ability Template Catalog (ATC) v1.0

## 目的

ATCは

```text
カード能力の構文パターン
```

を標準化する。

目的は

```text
能力文
↓
ATCテンプレート
↓
DSL生成
↓
DMVM実行
```

を実現すること。

---

# 全体構造

```cpp
struct FAbilityTemplate
{
    TemplateID;

    Name;

    Category;

    Parameters[];

    GeneratedDSL;

    GeneratedBytecode;
};
```

```

```

を検証する。

# v2で必要になるもの

ATC v1.0で扱えるのは、

```text
単文能力
```

です。

しかし実際のデュエマには

```text
相手のクリーチャーを1体破壊する。
そうしたらカードを1枚引く。

〜するたび、
〜なら、
〜するまで、
その後、
かわりに、
```

のような複文・依存構文が大量にあります。

そのため次は

# Ability Grammar Specification (AGS)

が必要になります。

役割は、

```text
ATC
=
単語辞書

AGS
=
文法
```

です。

ATCだけでは「単文」を理解できても、「能力文全体の意味構造」を理解できません。

AGSは能力文を抽象構文木（AST）へ変換するための正式文法仕様になります。

これはDMVM完成に向けて非常に重要です。

ここまで作った仕様群は、

```text
ルールを実行する仕組み
```

でした。

しかし実際のデュエマカードの90%以上は

```text
スピードアタッカー
W・ブレイカー
ブロッカー
侵略
革命チェンジ
G・ストライク
EXライフ
```

などのキーワード能力で記述されています。

つまりLLMが最終的に出力するカードJSONも

```json
{
  "KeywordAbilities":[
    "SpeedAttacker",
    "DoubleBreaker"
  ]
}
```

になるはずです。

そこで必要になるのが

# Keyword Ability Formalization Specification (KAFS) v1.0

です。

---

# 1. 目的

KAFSは全キーワード能力を

```text
公式テキスト
↓
意味論(Semantics)
↓
DSL
↓
Bytecode
```

へ変換するための標準辞書である。

---

# 2. KAFSの位置

```text
Card JSON
↓
KeywordAbilities
↓
KAFS
↓
Generated DSL
↓
DMVM
```

---

# 20. KAFS v2で追加すべきもの

v1.0で扱うべきなのは約150〜200種の主要キーワードです。

しかしデュエマ完全再現を目指すなら、次の段階では

```text
22,528枚すべてのカード能力
```

を

```text
キーワード
+
テンプレート能力
+
自由記述能力
```

へ分類する必要があります。

そのため次に必要なのは

# Ability Template Catalog (ATC)

です。

これは

```text
「相手のクリーチャーを1体破壊する」
「カードを1枚引く」
「◯◯なら〜する」
```

のような、数万枚のカードに共通する能力テンプレートを標準化する辞書になります。

実際には

```text
KAFS
+
ATC
```

で全カードの80〜95%程度をDSLへ自動変換できるようになるはずです。

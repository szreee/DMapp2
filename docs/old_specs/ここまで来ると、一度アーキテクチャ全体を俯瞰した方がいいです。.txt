# 現在の設計の問題

今まで作ったもの

```text
BCIP
ORIP
ORKB
Reasoner
DMQL
DSL
Compiler
DMVM
PRS
SBA
CELS
CCS
```

は個々では非常に良いです。

しかし実際のデュエマ裁定を再現しようとすると、

```text
カード能力
↓
DSL
↓
DMVM
```

だけでは足りません。

理由。

デュエマには

```text
総合ルール
↓
カード能力
↓
裁定
↓
例外裁定
```

という階層があるからです。

---

例えば

### 《母なる大地》

のようなカード。

カードテキストだけ見ると

```text
クリーチャーをマナへ
↓
山札から出す
```

です。

しかし実際には

```text
離れる順序

公開タイミング

シャッフルタイミング

置換効果との関係
```

など大量のルールが絡みます。

---

つまり本当に必要なのは

```text
Ability Engine
```

ではなく

```text
Rules Operating System
```

です。

## 次に本当に必要な設計

ここまで来ると、実は ORKB Reasoner より先に

```text
Ruling Intermediate Representation (RIR)
```

を定義した方が良いです。

なぜなら、

```text
総合ルール
公式Q&A
FAQ
判例
```

を全部そのままReasonerに食わせると管理不能になるからです。

コンパイラで言うと、

```text
自然言語
↓
AST
↓
IR
↓
Machine Code
```

の **IR** に相当するものがまだ存在していません。

そして22,000枚以上のカードと数万件規模の裁定を扱うなら、RIRはほぼ必須になります。

# Duel Masters Digital Rule Engine Specification v4.0

## (LLM Native Architecture)

# 1. 開発目標

本システムはデュエル・マスターズ紙環境の完全再現を目的とする。

以下を必須要件とする。

* サーバー権威
* JSONによるカード追加
* LLMによるカード生成
* カード毎の実装禁止
* DSL実行方式
* 非同期オンライン対戦対応

---

新ギミック追加時

Card JSON追加のみ。

RuleEngine改修を最小化すること。
v4でさらに追加したい要素

実はこの仕様でも、デュエマ完全再現にはまだ足りません。

次に必要になるのは、

Effect DSL v2
「そうしたら」
「その後」
「そうした場合」
「〜するかわりに」
「可能なら」
「〜するたび」
Query Language
「名前に○○を含む」
「コスト5以下」
「ドラゴンを持つ」
「このターン出た」
Continuous Layer System
MTGのレイヤーほどではないが、継続効果競合を解決する仕組み

です。

特にデュエマ完全再現の最大の難所は「ルールエンジン」ではなく、「カードテキストをDSLへ落とし込む仕様」なので、次のv5は Effect DSL v2 の設計になると思います。

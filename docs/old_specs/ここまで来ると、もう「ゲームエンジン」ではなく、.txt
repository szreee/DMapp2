ここまで来ると、もう「ゲームエンジン」ではなく、

```text
Duel Masters Card Operating System
```

の領域です。

そして実は、

```text
Validator
```

だけでは全く足りません。

例えば、

```text
このクリーチャーが出た時、
相手のパワー3000以下のクリーチャーを1体破壊する。
```

というカードがあったとします。

Validatorは

```text
構文OK
型OK
DSL OK
Bytecode OK
```

までしか保証しません。

しかし、

```text
対象が存在しない時

置換効果がある時

アンタッチャブルの時

同時誘発がある時
```

に本当に正しく動くかは別問題です。

そこで必要になるのが CCS です。

---

# Card Certification System (CCS)

## 目的

カード定義を

```text
登録可能
```

ではなく

```text
ゲーム投入可能
```

にする。

---

```text

無限ループ

未使用変数

常に真

常に偽

不正なQuery
```

```json

 "Node":"If",

}
```

Else到達不能

Warning

最重要。

---

自動で盤面を生成する。

---

例

```text

```

↓

```text

ケース2

ケース3
相手5体

```

# 3. Simulation Runner

DMVM上で実行。

---

```text
↓

↓

結果取得
```

---

# Card Test Framework (CTF)

CCSの実行基盤。

---

# テスト記法

YAML形式推奨。

---

例

```yaml
name: destroy_small_creature

setup:
  playerA:
    battlezone:
      - TestCard

  playerB:
    battlezone:
      - power: 3000
      - power: 5000

action:
  cast: TestCard

expect:
  destroyed:
    - power: 3000

  alive:
    - power: 5000
```

## 完全版を目指すなら最後に必要なもの

ここまで作るなら、さらに

```text
Official Ruling Knowledge Base (ORKB)
```

を設計したいです。

なぜならデュエマには、

* 総合ルール
* 公式Q&A
* 裁定変更履歴
* 特例裁定

が大量にあり、カードテキストだけでは再現できません。

実際の完全再現エンジンでは、

```text
Card JSON
+
Rule Engine
+
Official Ruling Knowledge Base
```

の3つが揃って初めて紙環境に近い挙動になります。

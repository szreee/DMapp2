これはデュエマ完全再現エンジンにおいて、OICSと並ぶ**最重要仕様の一つ**です。

実際、デュエマは他TCGと比較して

```text
カード = オブジェクト
```

ではありません。

むしろ

```text
複数カード
↓
1つのゲームオブジェクト
```

が頻繁に発生します。

例えば

* 進化クリーチャー
* 超無限進化
* 侵略
* 革命チェンジ
* オレガ・オーラ
* クロスギア
* ドラグハート装備
* タマシード
* フィールド
* 禁断＋封印

はすべて「複合オブジェクト」です。

そのためDMVMでは

# Object Composition System (OCS) v1.0

を独立仕様として定義するべきです。

---

# 1. 目的

OCSは

* 複数カードによる単一オブジェクト形成
* 構成要素管理
* 上下関係管理
* 効果継承
* 離脱処理
* 分離処理

を管理する。

# 現時点で不足している最大の仕様

ここまでで

* ETS
* ZTS
* OICS
* RRS
* EDDS
* IVCS
* CPRS
* ALDS
* TWSAS
* MRPS
* OCS

が揃います。

しかし、デュエマ完全再現において、実は最後の超巨大仕様がまだ残っています。

それは

# Keyword Ability Formalization Specification (KAFS)

です。

なぜなら22,000枚超のカードの大部分は、

```text
W・ブレイカー
T・ブレイカー
スレイヤー
ブロッカー
スピードアタッカー
侵略
革命チェンジ
G・ストライク
S・トリガー
ジャストダイバー
マッハファイター
EXライフ
```

などのキーワード能力で構成されています。

LLMがJSON化するときも、

```json
{
  "KeywordAbilities":[
     "SpeedAttacker",
     "DoubleBreaker"
  ]
}
```

のように出力する方が圧倒的に安定します。

そのためDMVM完成に向けて次は

**Keyword Ability Formalization Specification (KAFS)**

を定義する段階になります。これは「全キーワード能力をDSL/VMへ落とし込む辞書仕様」です。

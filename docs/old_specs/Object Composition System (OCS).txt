これはかなり重要です。

ここまで設計した

* ETS
* ZTS
* OICS
* RRS
* EDDS
* IVCS
* CPRS
* ALDS
* TWSAS

は、どちらかというと

```text
イベント
効果
状態
```

を扱う仕組みです。

しかしデュエマには、

```text
シールドブレイク
ガチンコ・ジャッジ
覚醒
禁断解放
超無限進化
龍解
侵略
革命チェンジ
```

のような、

**公式ルールで定義された「手続き（Procedure）」**

が大量に存在します。

これらは単一イベントではなく、

```text
Step1
↓
Step2
↓
Step3
↓
...
```

という複数段階の処理です。

そこで必要になるのが

# Multi-Step Resolution & Procedure System (MRPS) v1.0

です。

---

# 1. MRPSの目的

MRPSは

```text
公式ルール手続き
カード固有手続き
特殊召喚手続き
判定手続き
```

を統一的に管理する。

---


通常のDMVM命令は


```text

+
Window
+

```

# 26. v2で追加すべきもの

ここまででDMVMはかなり完成します。

ただし、紙デュエマ完全再現を目指すなら、次に必要になるのは

# Object Composition System (OCS)

です。

なぜかというとデュエマには

* 進化
* 超無限進化
* 侵略
* 革命チェンジ
* オレガ・オーラ
* クロスギア
* ドラグハート装備
* タマシード付与

など、

**「複数カードが1つのゲームオブジェクトになる」**

仕組みが大量に存在するからです。

実はここが、一般的なTCGエンジンでは最も実装が難しい領域の一つです。

現在のあなたのDMVM構想だと、

```text
OICS
=
カードの同一性

OCS
=
複合オブジェクトの構造
```

という分離が最も綺麗な設計になります。

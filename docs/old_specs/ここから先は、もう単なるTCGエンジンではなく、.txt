現在のORKBは

```text

Ruling

```

を保存できます。

しかし保存しているだけです。

例えば

```text

```

という置換効果。

ORKBには

```json
{
  "Pattern":"DestroyReplacement",
  "Result":"NotDestroyed"
}
```

が保存されている。

```text

```

```text

```

の誘発もある。

つまり推論が必要。

# 最終形

完成すると。

```text
ORIP
 ↓
ORKB
 ↓
Reasoner
 ↓
Derived Rules
 ↓
CCS
 ↓
Certified Knowledge
 ↓
DMVM
```

になります。

---

ただし、ここで一つ設計を見直した方がいい点があります。

今の構想だと

```text
DMVM
↓
ORKB
↓
Reasoner
```

となっていますが、実際のゲーム実行時に毎回Reasonerを呼ぶと重すぎます。

私なら最終的に

```text
ORIP
↓
ORKB
↓
Reasoner
↓
Compiled Ruling Rules
↓
Rule Bytecode
↓
DMVM Runtime
```

にします。

つまり **Reasonerは開発時・インポート時に使うコンパイラであり、対戦中は推論済みルールだけを実行する** 設計です。

これは22,000枚超のカードと大量の裁定を扱うなら、ほぼ必須だと思います。
